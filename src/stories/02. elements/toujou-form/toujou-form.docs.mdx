# &lt;toujou-form&gt;
The toujou form element (with the class `toujou-form`) render form and form elements on the page. It can be created in a couple different ways, but most probably it will be created using the "Forms" module on the BE and then placed on a page using the "Mail Form" plugin.

## Markup example
```html
<form class="toujou-form" id="testform">
    <toujou-input-group class="toujou-input-group" input-group-type="text">
        <label class="input-label" for="testform-1000091-text-1">Ihr Name</label>
        <toujou-form-description class="toujou-form-description">Enter your name so we can print out the documents correctly.</toujou-form-description>
        <input
            placeholder="Max Mustermann"
            data-pristine-required-message="The given subject was empty."
            class="input input--text"
            id="testform-1000091-text-1"
            type="text"
            name="tx_form_formframework[testform-1000091][text-1]"
            value=""
            required="required"
            ${disabledAttribute}>
        <toujou-form-error class="pristine-error toujou-form-error">The given subject was empty.</span>
    </toujou-input-group>
</form>
```

<a name="toujou-input-group"></a>

## 1. &lt;toujou-input-group&gt;
This is a wrapper element for all input types. It helps us with styling, validation and layout. It may have different attributes:

| Attribute          | What is it?                                                                 | Required |
|--------------------|-----------------------------------------------------------------------------|:--------:|
| `class`            | It must have at least the `toujou-input-group` class                        |   yes    |
| `input-group-type` | Defines the type of child input. Ex: `input-group-type="text"`              |   yes    |
| `is-required`      | Added to the group if the input is required. Will add `*` to the label      |   yes    |
| `has-success`      | Added to the group when validation passes                                   |    no    |
| `has-error`        | Added to the group when validation fails                                    |    no    |
| `is-disabled`      | Added to the group when the child input is disabled (disables whole group!) |    no    |



The `toujou-input-group` also accepts attributes to be placed inside the `toujou-row` element. Each of these attributes takes a value between 1 and 12, for the number of columns it should span inside the row.

| Attribute             | What is it?                                     |
|-----------------------|-------------------------------------------------|
| `input-group-span-xs` | Number of columns on extra small devices        |
| `input-group-span-sm` | Number of columns on small devices ( > 480px )  |
| `input-group-span-md` | Number of columns on medium devices ( > 840px ) |
| `input-group-span-lg` | Number of columns on large devices ( > 1000px ) |

Example:
```html
<toujou-input-group
    class="toujou-input-group"
    input-group-type="text"
    input-group-span-xs="12"
    input-group-span-sm="6"
    input-group-span-md="4"
    input-group-span-lg="4">
    ...
</toujou-input-group>
```

<a name="toujou-form-row"></a>

## 3. &lt;toujou-form-row&gt;
This element can render child `toujou-input-groups` as a grid. The grid has 12 columns.

<a name="toujou-form-description"></a>

## 3. &lt;toujou-form-description&gt;
This element renders the description for the input (if one is available)

<a name="toujou-form-error"></a>

## 4. &lt;toujou-form-error&gt;
This element renders the validation error for the input when the validation fails

## 5. Examples
Here you can find markup examples for the different types of inputs

<a name="toujou-input-example-text"></a>

#### 5.1 Text inputs
(Also other input types, like `url`, `telephone`, `number`, ... See story)

```html
<toujou-input-group
    class="toujou-input-group"
    input-group-type="text">
    <label class="input-label" for="testform-1000091-text-1">Ihr Name</label>
    <toujou-form-description class="toujou-form-description">Enter your name so we can print out the documents correctly.</toujou-form-description>
    <input
        placeholder="Max Mustermann"
        data-pristine-required-message="The given subject was empty."
        class="input input--text"
        id="testform-1000091-text-1"
        type="text"
        name="tx_form_formframework[testform-1000091][text-1]"
        value=""
        required="required">
    <toujou-form-error class="pristine-error toujou-form-error">The given subject was empty.</span>
</toujou-input-group>
```
